## 产品概述

为 BroadcastChannel 项目增加广告关键字屏蔽功能，自动过滤包含预设关键字的帖子内容。

## 核心功能

- 通过环境变量配置屏蔽关键字列表（逗号分隔）
- 检测帖子标题和文本内容是否包含关键字
- 自动屏蔽匹配的帖子，不展示在页面中
- 保持对 Vercel、Cloudflare、Netlify、Node.js 的兼容性

## 技术栈

- 框架: Astro (Hybrid 模式)
- 语言: JavaScript (与现有项目保持一致)
- 配置: 环境变量方式配置关键字列表

## 实现方案

### 策略概述

在现有的帖子过滤逻辑中增加关键字检测步骤，通过环境变量 `BLOCK_KEYWORDS` 配置屏蔽词列表，使用大小写不敏感的字符串匹配检测帖子标题和文本内容。

### 关键技术决策

1. **环境变量格式**: 使用逗号分隔的字符串（如 `BLOCK_KEYWORDS=广告,推广,加微信`），简单直观且易于维护
2. **匹配范围**: 检测 `post.title` 和 `post.text` 两个字段，覆盖帖子主要文本内容
3. **匹配方式**: 使用 `String.prototype.includes()` 配合 `toLowerCase()` 进行大小写不敏感匹配
4. **过滤位置**: 在现有的 filter 链中添加，保持代码简洁

### 性能考虑

- 关键字检测在帖子解析后、缓存前执行，不会增加额外的网络请求
- 使用简单的字符串匹配，时间复杂度为 O(n*m)，其中 n 为关键字数量，m 为文本长度
- 关键字数量通常较少（预期 10-50 个），性能影响可忽略

## 实现细节

在 `src/lib/telegram/index.js` 中：

1. 添加关键字检测函数：

```javascript
function shouldBlockPost(post, keywords) {
  if (!keywords || keywords.length === 0) return false
  const content = `${post.title || ''} ${post.text || ''}`.toLowerCase()
  return keywords.some(keyword => content.includes(keyword.toLowerCase()))
}
```

2. 在 `getChannelInfo` 函数中读取环境变量并过滤：

```javascript
// 在现有 filter 逻辑中添加关键字过滤
const blockKeywords = getEnv(import.meta.env, Astro, 'BLOCK_KEYWORDS')
  ?.split(',')
  .map(k => k.trim())
  .filter(Boolean) || []

const posts = $('.tgme_channel_history .tgme_widget_message_wrap')
  ?.map((index, item) => getPost($, item, { channel, staticProxy, index }))
  ?.get()
  ?.reverse()
  .filter(post => 
    ['text'].includes(post.type) && 
    post.id && 
    post.content && 
    !shouldBlockPost(post, blockKeywords)
  )
```
